Module 5 - Compliance Month 2

What you'll need:

    ➢ Module 2 input file

Summary

Within this module we will understand how actions taken within AEWS need to be reflected in payroll, we will also demonstrate how to use the ‘data correction’ action when you have uploaded incorrect data into AEWS.

During this month ABC employer has several amendments to their employee records:

➢ Emma Jones decided to temporarily reduce pension contributions

➢ Peter Clarke left employment

➢ Sharon Kitton joined the company


All of the above actions would be identified in the file prior to the upload in to AEWS. The employees who decided to reduce payments would have reduced contribution amounts.

Internal Steps ABC Employer took

➢ Followed their internal timeline

➢ Uploaded month 2 into AEWS

➢ Uploaded the payment to provider file transfer

Steps for you to complete
  1. Log into AEWS Test Site

  2. Select 'Start Compliance Check'

  3. Select 'No' to upload Module 5 (This file contains the employee changes mentioned above)

  4. Select the next pay reference period (PRP) (month 2)

  5. Select ‘Start Compliance Check’

  6. Complete the following actions in AEWS:



  7. Save the activity files within the AEWS folder

  8. Reload the amended Module 5 file for the same PRP and save the Payment file.

We have now successfully run Compliance Month 2 and have observed how there is a recurring link between payroll and AEWS. In some instances payroll will produce actions for AEWS (temporary reduction in payments etc) and some instances

AEWS will generate actions in payroll (increasing contributions etc). This process needs to be documented so that changes between the systems can be done between the payroll closing and the payroll being run.

Key learning outcome

➢ The 3 data items that you may have to change in payroll after using AEWS are employer & employee contributions and existing member indicator
